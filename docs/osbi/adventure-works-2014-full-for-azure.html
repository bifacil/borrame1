Adventure Works 2014 Full for Azure (BACPAC)

OSBI
martes, 2 de febrero de 2016

Desde la primera versión de Crono -hace 6 años ya- hemos utilizado una base de datos de ejemplo sobre una cadena de librerías (BI_BIBLIO). Esta base de datos contiene un sencillo modelo de ventas y nos ha permitido mostrar adecuadamente la funcionalidad básica de nuestro producto (ventas por fecha, por producto, por tienda, y por cliente...). Sin embargo, Crono ya es un producto consolidado, robusto, y tan potente como cualquiera, y esta demo se quedaba corta y ya no nos permitía mostrar adecuadamente todas las virtudes de nuestro producto...

Los datos de las empresas son complejos y variopintos. De hecho, esto mismo es lo que siempre he criticado de los cubos multidimensionales o los metadatos de otras herramientas de visualización... que utilizan unos modelos demasiado sencillos que no reflejan los casos reales con los que se pelean los usuarios de negocio...

Por todo ello, en Crono hemos decidido construir un nuevo catálogo Business Intelligence utilizando una base de datos más compleja y hemos optado por utilizar el conocido ejemplo de Microsoft SQL Server: Adventure Works 2014 Full. Esta base de datos tiene información de ventas, de RRHH, de compras, de producción... por lo que cubre perfectamente los objetivos que estamos buscando...

Sin embargo, sorprendentemente, nos ha costado encontrar esta base de datos lista para su importación en SQL Azure. Efectivamente, o no existe o no hemos sabido encontrar el fichero BACPAC con la estructura y los datos de esta base de datos. Hemos encontrado versiones "Lite" que tenían modelos y volúmenes de datos ridículamente pequeños... pero no la versión "Full".

En fin, que al final nos hemos tenido que arremangar y preparar nosotros mismos este fichero BACPAC para importar la base de datos en Azure.... Si alguna vez necesitas estos datos, espero que Google sepa que están aquí:

Adventure Works 2014 Full for Azure BACPAC

Para preparar esta base de datos hemos tenido que suprimir algunas funcionalidades que actualmente no se soportan en SQL Azure. Básicamente: propiedades extendidas (MS_Description...), tipos XML personalizados, y algunas configuraciones para la replicación de la base de datos. Las tablas, vistas, datos, relaciones se han exportado todas sin problemas.

Espero que el próximo que lo necesite lo encuentre con mayor facilidad.

Mientras tanto, nosotros seguiremos construyendo el catálogo y los ejemplos. Confío en que esté disponible a finales de esta semana. Keep tuned.

In English:

We needed to import the Adventure Works 2014 Full database to Azure, but we were not able to find the BACPAC file. Finally, we export the database for ourselves (we had to delete some unsupported features: extended propoerties, XML datatypes, replication config, etc). Everything else is ready for you to use:

Adventure Works 2014 Full for Azure BACPAC

Hope it helps.
